Before starting, you need to configure your input device. Commonly this means choosing an input device and adjusting its volume in the mixer.

After doing that, start NOOT. Usually there is no need for changing the default options.

Click on the Start/Stop button to start the tuner.

If you already know the note you are tuning and do not need to discover it, you should set the Note and Octave option (see the section called “Note selector” and the section called “Octave selector”). This will make note detection easier.

The main window

Here is an explanation of all the controls in the main window.

Start/Stop button

This button will start or stop the note detection. If another application need to open the input device, you can temporarily stop NOOT and restart it later, without closing its window.

'Detected note' box

This box is made of two parts. The tuning indicator will show how much the detected note is far from the expected note. Its colour varies according to whether the pitch is higher (blue) lower (red) or quite right (green). If the note is perfectly tuned, the indicator should be under the black line.

The lower part of the box shows the following information:

Note

The detected note. This value depends on whether you have set the note and the octave in the control panel below. If they are note set to Auto, the note displayed will be exactly the choosen one.

Offset

The offset from the expected pitch, in cents (1 cent = 0.01 semitones)

Frequency

The detected frequency in Hertz. This information is useless to most of the users.

Octave selector

Selecting the right octaves improves the precision of the detection and the readability of the indicator. When the octave is set to Auto, the program guesses it, sometimes leading to a note detection error.

With the standard used the middle C is in octave 3.

Sometimes the right octave is not the best choice, since in some cases (especially with extreme notes) the base frequency is too soft to be detected precisely. If the detection is not precise enough (i.e. the indicator glitches or vibrates too much) you can safely select an octave lower or higher than the original one. Remember that if you set the octave in the control panel, the detected octave may not correspond to that of the note. (For other advice see Chapter 3, Advanced usage).

Note selector

Like for the octave, specifying the note oftens improves the precision, but in most cases you should not need it.

'Listen' button

Click on this button to listen to the frequency of the note selected.

Temperament selector

With NOOT Instrument Tuner you can choose among a variety of temperaments. The explanation of the temperaments listed is not a subject of this manual.

Transpose

Use this control to transpose the input. It can be a decimal number.

The frequency displayed above will not be altered, but the note and/or the offset in cents may be.

Window size

This option affects the note detection algorithm. It represents the number of samples on which the FFT and autocorrelation (or continuous powr spectrum) are performed.

The optimal window size depends on the refinement algorithm selected. Consider however that in any case a bigger window implies a more accurate measure, at the cost of reducing the reactivity of the program and increasing the CPU usage.

If the default algorithm is set (Continuous power spectrum), the window size can be set at any value (at least 1/5 of the sample rate). If the alternate algorithm (Autocovariance) is selected, then when the window size is too little, the precision constraint might not be satisfied, and therefore no result is shown. The optimal window size in the latter case is between 16384 and 32768.

Threshold

This option is used to start the detection only if the input is loud enough, in order to avoid glitches due to the background noise. The threshold is displayed graphically as a thin black line in the volume meter.

A threshold of 0 dB means that the detection will never start, while a threshold of 90 dB (-90 dB, actually) will allow any input to be processed.

Expected precision

The lower is this number, the higher will be the precision. It represents the theoretical error in thousandths of semitone on the detected frequency.

Lower values also mean slower response and CPU usage.

Frame rate

This number represents the maximum number of updates per second. If it is set to a too high value, the program might not fulfill the request or even slow down.

Status bar

The status bar is composed of three boxes. The first contains information about the current status (Listening, Paused, etc.).

The second box contains two numbers: sampling frequency and clock correction factor (see the section called “Clock correction”).

The third box contains a volume meter. The thin black line represents the current threshold. Note detection happens only when the input volume is higher than the threshold. The volume meter turns red when possible clipping is detected. In case, the user should lower the volume, as clipping alters the frequency detection.

The menus

Devices

This menu contains options to select the input and output devices.

Other options

Through this menu you can change the current sampling rate and clock correction. See Chapter 3, Advanced usage for detailed instructions.

Change sample rate

This menu item opens a window listing some of the possible sample rate for the device selected. Sample rate should be as low as possible if the “Continuous power spectrum” refinement algorithm is selected, as high as possible otherwise.

Clock correction

The clock correction factor is a number that multiplies the detected frequency. Set this option if your sound card has an inaccurate clock. Clock correction can be set as a coefficient or in terms of cents (hundredths of a semitone). (See the section called “Clock correction”.)

Window type

This is a rather techical option. It should always be Hanning, unless you have some serious reason to change it.

Refinement algorithm

This options define the algorithm use to increase the precision of the frequency estimated in the first step.

None

Don't perform any refinement (useful only for testing, it gives very bad results)

Autocovariance

The old algorithm (up to version 0.2.x)

Continuous power spectrum

The new algorithm, faster and more reliable with extreme notes.

The default is Continuous power spectrum.
